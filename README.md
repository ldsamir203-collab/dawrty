# دورتي — منصة تنظيم دوريات eFootball

نموذج أولي (MVP) مبني بـ React + Vite + Tailwind CSS.

## 1) التشغيل محلياً

يلزمك Node.js مثبت على جهازك (نسخة 18 أو أحدث). حمّله من: https://nodejs.org

بعدين افتح Terminal / Command Prompt جوا مجلد المشروع وسوي:

```bash
npm install
npm run dev
```

رح يفتح الموقع على `http://localhost:5173`.

## 2) بناء نسخة الإنتاج

```bash
npm run build
```

هاد بيولد مجلد `dist/` فيه كل ملفات الموقع الجاهزة للنشر (HTML, CSS, JS).

## 3) النشر (رفع الموقع على الإنترنت)

أسهل طريقة هي **Vercel** أو **Netlify** (فيهم خطة مجانية):

### الطريقة الأسهل — بدون سطر أوامر (Netlify Drop)
1. روح لـ https://app.netlify.com/drop
2. اسحب مجلد `dist/` (بعد ما تعمل `npm run build`) وفلّته بالصفحة
3. خلص! رح يعطيك رابط شغال فوراً (مثال: `random-name.netlify.app`)

### الطريقة الاحترافية — عن طريق GitHub (موصى فيها لأي تحديثات مستقبلية)
1. ارفع المشروع على GitHub (سوي repo جديد وادفع الكود)
2. روح لـ https://vercel.com أو https://netlify.com وسجل دخول بحساب GitHub
3. اختار "Import Project" واختار الـ repo تبعك
4. اترك الإعدادات الافتراضية (Vite بيتعرف عليها تلقائياً) واضغط Deploy
5. من الآن وصاعداً، أي تعديل بتعمله وترفعه على GitHub بينشر تلقائياً

## 4) ربط دومان خاص فيك

1. اشتري دومان من أي مزوّد (Namecheap, GoDaddy, Google Domains...) — عادة 10-15$ بالسنة
2. بلوحة تحكم Vercel أو Netlify، روح لإعدادات المشروع → **Domains** → **Add domain**
3. حط اسم الدومان تبعك (مثلاً `dawrty.com`)
4. المنصة رح تعطيك سجلات DNS لازم تضيفها (عادة نوع `A` أو `CNAME`)
5. روح للوحة تحكم الدومان (عند المزوّد يلي اشتريت منه) وضيف هاي السجلات بقسم DNS
6. استنى من ساعة لـ 24 ساعة (انتشار DNS) وبعدها الموقع رح يشتغل على دومانك مباشرة، مع شهادة SSL (https) مجانية تلقائياً

## ملاحظة مهمة

هاد النموذج حالياً بيخزن البيانات بالذاكرة فقط (React state) — يعني أي دورة أو لاعب بتسجله بينمسح لما تسكر أو تحدّث الصفحة. هاد طبيعي لمرحلة الـ MVP/العرض. لما تكون جاهز، الخطوة الجاية هي ربط قاعدة بيانات حقيقية (متل Supabase أو Firebase) عشان البيانات تضل محفوظة لكل الزوار.
